Aula 33 - Formulários

    Nesta aula aprenderemos sobre formulários, uma parte muito importante de websites. Porém é um tópico muito extenso, caso queira se aprofundar no assunto clique aqui.

    form

    A tag <form></form> é utilizada para criar um formulário. Dentro dela usamos vários outras tags que tem a função de receber informações fornecidas pelo usuário.

    A tag form possui um atributo muito importante: action. Nele colocamos o endereço da página web que receberá os dados do formulário. Porém essa página faz parte do backend, que será responsável por validar, tratar e salvar esses dados. Existem alguns serviços que fazem isso para nós, sem precisarmos criar o novo próprio servidor web, como por exemplo o formspree.

    name, id e value

    As tags usadas dentro dos formulários podem receber diversos atibutos, os trê mais importantes são name, id e value.

    Value é o atributo que guarda o valor de cada campo. Pode ser usado com um valor inicial, mas geralmente o usuário que fornece o valor final.

    Id é um atributo que já vimos, além de todas utilidades que já conhecemos, ele é usado para criar labels, veremos isso mais adiante.

    Name é usado para identificar um campo quando enviamos o formulário, geralmente é mais importante para o desenvolvedor do backend.

    input

    Provávelmente a tag mais usada em formulários. Ela possui um atributo muito importante: type.

    Dependendo do valor usado em type, a tag input se comporta de maneira diferente, podendo ser uma entrada de texto, um botão de envio, um seletor de datas ou cores, um seletor de uma opção, um seletor de várias opções, entre outros.

    Tome cuidado antes de usar um type, alguns deles não são suportados por todos navegadores.

    Alguns valores de type são substituídos por type="text" em navegadores que não surportam aquele type. Porém é sempre bom utilizar o tipo certo nos seus inputs, isso cria algumas facilidades para o usuário, como por exemplo: relembrar valores digitados em inputs do mesmo tipo (mesmo em outros sites), validação automática, teclado específico em mobile.

    select e option

    Uma das tags usadas dentro da tag form é a tag <select>, que é usada para que o usuário selecione um valor dentro de algumas opções. As opções são escritas usando a tag <option> dentro da tag select.

    textarea

    Esta tag é usada para inserir textos de multiplas linhas.

    button

    Esta tag é usada para criar um botão.

    progress

    É uma barra de progresso.

    label

    Esta tag é usada para dar um rótulo a uma outra tag dentro do formulário. Ela possui o atributo for que deve receber o mesmo valor que o id da tag que queremso rotular. Se clicada ela dá foco ao elemento rotulado.

    fieldset

    Esta tag é usada para agrupar elementos dentro de um form. Ela desenha uma caixa em volta dos elementos agrupados.

    legend

    Esta tag define uma legenda para um fieldset. Útil para acessibilidade de pessoas com deficiência visual.

    Estilos

    Quando queremos usar estilos CSS em um elemento de um formulário é muito comum utilizarmos alguns seletores diferentes. Por exemplo o seletor por atributo: input[type=text], seleciona todos elementos com a tag input e o atributo type igual a text.

    Outro seletor muito utilizado é o pseudo-seletor focus, ele é usado quando o elemento é focado (selecionado).

    Também é muito usado o pseudo-seletor checked, junto com o seletor + que seleciona elementos imediatamente depois.

    Veja o exemplo:

    No index.html:

              
            <input type="text" value="" id="nome" placeholder="Nome" />
            <input type="checkbox" name="check" id="check">
            <div id="surpresa">Surpresa!!!</div>

    No style.css:

              
            input[type="text"] {
              display: block;
              height: 20px;
              font-size: 18px;
              width: 20%;
              border: none;
              border-bottom: 2px solid rgb(27, 107, 212);
              transition: 0.2s;
            }

            input[type="text"]:focus {
              width: 50%;
              outline: none;
            }

            #surpresa {
              display: none;
            }

            input[type=checkbox]:checked+#surpresa {
              display: block;
            }

    Validações

    Quando usamos um tipo em um input, por exemplo type="email", automaticamente ocorre uma validação daquele campo, ou seja, verifica se o que foi digitado tem um formato de e-mail.

    Além disso podemos usar alguns atributos para criar mais validações.

    required

    O atributo required, pode ser usado, sem nenhum valor, apenas o nome do atributo, para indicar que aquele elemento é obrigatório.

    minlength e maxlength

    Esses atributos determinam o número mínimo e máximo de caracteres de um input.

    min, max e step

    Os atributos min e max determinam o mínimo e o máximo de um valor em um input com type="number". O atributo step define de quanto em quanto o número aumenta quando usamos as setas do lado do input.

    CSS :valid e :invalid

    As pseudo-classes valid e invalid são usadas para estilizar inputs válidos e inválidos respectivamente.

    Clique aqui para saber mais sobre validação de formulários.

    Vejamos agora um exemplo de código que mostra as propriedades vistas acima:

    No index.html:

              
            <label for="nome">Nome:</label>
            <input type="text" value="" id="nome" required minlength="2" maxlength="15" />
            <label for="decada">Década de Nascimento:</label>
            <input type="number" name="decada" id="decada" value="1990" min="1890" max="2010" step="10">

    No style.css:

              
            input {
              display: block;
              margin-bottom: 15px;
            }

            #nome:invalid {
              border: 2px solid red;
            }

            #nome:valid {
              border: 2px solid green;
            }

    Eventos

    Uma utilidade muito grande dos inputs são os eventos relacionados a eles. Os mais usados são: change, input e submit. Veja o exemplo abaixo, que faz algo simples, escreve em um h1 o que o usuário escrever no input.

    No index.html:

              
            <input type="text" id="texto" />

            <h1 id="cabecalho"></h1>

    No app.js

              
            let texto = document.getElementById('texto');
            let cabecalho = document.getElementById('cabecalho');
            texto.addEventListener('input', function () {
              cabecalho.textContent = texto.value;
            });
