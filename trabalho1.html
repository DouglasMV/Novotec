Trabalho 1 - Website de um Filme

Neste trabalho faremos um website para um filme, contendo informações, fotos e vídeos sobre o mesmo. Usaremos apenas HTML e CSS.

Este trabalho pode ser feito individualmente ou em duplas.

É proibido copiar e colar qualquer código neste trabalho. É permitido sim pesquisar, perguntar ao professor e a outros colegas. É importante que vocês entendam o que cada linha de código digita faz, apenas copiar de um lugar não ajuda no aprendizado.

Utilizaremos como modelo esse website.

O website deve conter obrigatoriamente as seguintes seções:

  • Início: é a seção inicial do site, o primeiro contato do usuário com o website.
  • Sobre: um lugar onde podemos encontar informações sobre o filme.
  • Trailer: um lugar onde podemos assistir ao trailer do filme.
  • Crítica: uma crítica do filme, pode ser escrita por você mesmo ou copiada de outro site (coloque o nome do autor e um link para o site original)
  • Estrelas/Filmes: um lugar onde temos pelo menos 3 pessoas ligadas ao filme, ou 3 filmes relacionados.
  • Galeria: aqui devemos ter fotos do filme.

A seção de Estrelas/Filmes deve ter links para outras páginas do website sobre aquelas pessoas ou filmes. (veja o modelo)

Essas são as partes obrigatórias do website, caso você deseje pode acrescentar mais conteúdo. Porém faça primeiro o obrigatório, depois pense em acrescentar algo a mais, pois o tempo sempre é um fator.

O website deve ser feito dentro de um repositório no GitHub e publicado utilizando o GitHub Pages. Consulte a aula 7 para lembrar como isso é feito.

O website deve ser responsivo (ser bem apresentável em todos tamanhos de tela) e "mobile first" (os estilos para dispositivos com telas menores são carregados primeiro).

Lembre-se de utilizar as principais meta tags. Em especial a tag que indica o(s) autor(es) do website, coloque seu nome (nome dos dois integrantes da dupla).

Os estilos CSS devem ser modularizados, ou seja, divididos em vários arquivos. Como aprendemos na aula 23. Veja o repositório do modelo para ter uma ideia de como dividir os estilos.

Os seguintes aspectos serão levados em consideração para composição da nota:

  • Experiência do Usuário: o website deve ser bonito, fácil de usar e entender e usar, funcionar corretamente.
  • Responsividade: o wesite deve ser bonito em todos tipos de dispositivos e tamanhos de tela.
  • "Limpeza" do código: o código deve ser bem organizado. Os estilos CSS devem ser modularizados em arquivos pequenos. Devem existir pastas para cada tipo de arquivo (imagens, CSS, HTML, etc).
  • Participação: O(s) aluno(s) devem se mostrar interessados em aprender e buscar soluções para os problemas. Devem pesquisar, perguntar ao professor, perguntar aos colegas, etc. Ajudar os colegas também vale nota.
  • Originalidade: fazer um website igual ou parecido com o modelo é fácil. Trazer algo de novo, um diferencial, uma maneira diferente de fazer o website, criatividade. Mas faça coisas mais da forma mais simples, sem criar dificuldades desnecessárias. E lembre-se que tudo deve ter um motivo.
